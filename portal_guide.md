# Как пользоваться порталами (Multiverse Portals)

Краткий гайд для игроков.

Есть **два режима**: локальные (шерсть, внутри сервера) и межсерверные (`[Multi]` / `[To]` / `[Pair]`).

**Языки:** [English](portal_guide.en.md) · [中文](portal_guide.zh.md) · [Русский](portal_guide.md)

---

## Локальные порталы (шерсть)

Как ColorPortals — телепорт **в пределах этого сервера** (можно в другой мир).

1. Рамка из **шерсти одного цвета**, контур примерно **3×4**:
2. На верхний средний блок шерсти — **настенная табличка**
3. Внутри: **нажимная плита** (и по желанию кнопка под табличкой)
4. **1-я строка** — имя, **2-я** — канал (число `0`–`9999`)

Порталы с **тем же цветом и каналом** связываются в кольцо: A→B→C→A. Два портала = туда-обратно.

Встань на плиту или нажми кнопку. ПКМ по табличке — инфо. Список: `/mvp local list`.

## Межсерверные порталы

## Собрать портал

1. Рамка как у адского портала — проём **2×3** (воздух), вокруг любой твёрдый блок:

```
O O O O
O . . O
O . . O
O . . O
O O O O
```

`O` = рамка, `.` = воздух.

2. На рамку повесь **табличку**.
3. Рядом (1–2 блока) положи **нажимную плиту**.
4. На **1-й строке** таблички напиши тип (регистр не важен).

Рамка нужна для красивой «материи» внутри. Для работы достаточно таблички + плиты.

## Типы на 1-й строке

| 1-я строка | Что делает |
|------------|------------|
| `[Multi]` / `[mvp]` / `[MVP]` / `[portal]` / `[Random]` | Случайный сервер (линк **закрепляется**) |
| `[To]` | Конкретный сервер: **IP + порт** (2-я / 3-я строка) или id из каталога |
| `[Pair]` | Парный портал туда-обратно |

Скобки можно не ставить: `mvp`, `portal`, `Multi` — тоже ок.

### Связать два своих сервера (админ)

Подробная инструкция в [README.ru.md](README.ru.md#связать-два-своих-сервера-туда-обратно). Кратко:

1. **Лучший вариант:** на Alpha табличка `[Pair]` → скопировать код → на Beta `[Pair]` + код во 2-й строке.  
2. **Через IP:** `Portal` / `1.2.3.4:25565` (или только адрес, если порт `25565`) — каталог не нужен.  
3. **Через id:** Alpha `[To]` / `beta`, Beta `[To]` / `alpha` (id из `/mvp info`).  
4. Для id оба сервера должны быть в публичном каталоге [mp.mvse.ws](https://mp.mvse.ws/) (нужны `accept-transfers=true` и адрес, доступный из интернета; `public-host` обычно берётся из `server-ip`).

English: [README.md](README.md#link-two-of-your-servers-round-trip) · 中文: [README.zh.md](README.zh.md#两台服互通往返) · Русский: [README.ru.md](README.ru.md#связать-два-своих-сервера-туда-обратно)

### Random = sticky

1. Ставишь `[Multi]` → белые частицы → портал **привязывается к одному серверу**.
2. Пока табличка на месте — всегда туда (и после рестарта сервера тоже).
3. Хочешь другой сервер — **снеси табличку** и поставь `[Multi]` заново.
4. Если цель временно лежит — линк **не меняется**; попробуй позже или пересоздай табличку.

## Как войти

Встань на **плиту** у таблички. Подожди заряд — тебя перекинет.

## Согласие на one-way (`/mvp ready`)

Есть два вида переходов:

- **С возвратом** — парный `[Pair]` или сервер с тем же плагином. Заходишь **свободно**, согласие не нужно.
- **В одну сторону** — случайный публичный сервер из сканера. Обратно этим порталом не вернуться.

Перед one-way **один раз** разреши:

```
/mvp ready
```

или просто в чат:

```
mvp ready
```

Снять согласие:

```
/mvp ready off
```

или в чат: `mvp ready off`.

## Входящий трафик (для админа)

На сервере-приёмнике можно отсеивать приток:

```
/mvp ingress           — лимиты
/mvp deny add Nick …   — в бан порталов
/mvp rep Nick -50      — штраф к репутации
```

Подробнее: [README.ru.md](README.ru.md).

## Нагрузка на сервер (для админа)

Плагин **не «тяжёлый» постоянно**, но и не «нулевой» как маленький utility. В фоне почти всё async; **пики** — при создании `[Multi]` (Scan… + пробы), при зарядке на плите и если много порталов с визуалом.

| Что | Когда | Насколько |
|-----|--------|-----------|
| Сканеры (MineScan + Cornbread) | раз в ~1–2 мин | HTTP + SQLite — **TPS почти не трогают** |
| Bind при создании `[Multi]` | один раз | пробы по сети в фоне — **до ~90 с** |
| Переход после bind | стоишь на плите | ~2 с charge + transfer, **без долгого поиска** |
| Частицы «материи» | каждые 0.5 с | только если игрок **ближе ~20 блоков** |
| Локальные wool-порталы | нажатие плиты | **очень лёгко** |

**Слабый VPS (1–2 GB RAM):** нормально как **сервер-приёмник** с 1–2 привязанными `[Multi]`. Не ставь десяток порталов с эффектами и не пересоздавай таблички без нужды.

**Урезать нагрузку** — в `config.yml`:

```yaml
scanner:
  refresh-seconds: 180
  sample-count: 20
  max-attempts: 10
  bind-search-seconds: 45

effects:
  matter:
    enabled: false      # или particles: false

registry:
  enabled: false   # так и оставляй — открытый каталог по HTTPS
```

Самый лёгкий режим: только **PAIR** или только **локальная шерсть**. Подробная таблица — [docs/TECHNICAL.md](docs/TECHNICAL.md#performance--weak-servers).

## Вещи

- На сервер **с плагином** (пара / сеть) — вещи могут уехать и приехать (зависит от настроек `/mvp items`).
- На **публичный one-way** — вещи **остаются у тебя** в инвентаре (их не забирают).

## Версии

В игре: **`/mvp version`** — какая версия стоит и какая свежая на mp.mvse.ws.

Плагин не кидает на сервер, куда твой клиент не зайдёт. Если целей нет — «нет совместимых серверов».

Скачай jar с [mp.mvse.ws](https://mp.mvse.ws/) — в имени файла указана версия (`MultiversePortals-1.1.14.jar`).

## Админу: `accept-transfers` и каталог

В **`server.properties`** (не в плагине):

```properties
accept-transfers=true
```

Без этого Minecraft **отклоняет** вход по Transfer — гости с других миров не зайдут. Плагин тогда работает в **локальном режиме** (шерсть, Pair/`[To]` по IP) и **не публикуется** на mp.mvse.ws.

Адрес для Transfer плагин берёт из `server-ip` / `server-port` сам. В `config.yml` `public-host` нужен только если игроки заходят по другому домену. Проверка: `/mvp settings`.

## Частые проблемы

| Сообщение | Что делать |
|-----------|------------|
| Нужен `/mvp ready` | Напиши `/mvp ready` или `mvp ready` в чат |
| Нет совместимых серверов | Подожди / другая версия клиента; админ: `/mvp scanner` |
| Не удалось найти живой сервер | Попробуй плиту ещё раз |
| Парный портал сломан | Нужно связать пару кодом `[Pair]` |
| Сервер не в каталоге | `accept-transfers=true`, открытый IP/домен, не LAN; см. `/mvp settings` → catalog |

```
/mvp scanner        — сколько серверов в пуле (инфо)
```
